# Contributing to hulud-scan

First off, thank you for considering contributing to `hulud-scan`. It's people like you that make open source such a great community.

## How to Contribute

There are many ways to contribute, from writing documentation, to reporting bugs, to submitting pull requests with new features or bug fixes.

If you have a new feature or a bug fix, please make sure to:

1.  **Fork the repository** on GitHub.
2.  **Create a new branch** for your changes.
3.  **Write tests** for your changes (if applicable).
4.  **Ensure all tests pass**.
5.  **Submit a pull request** with a clear description of your changes.

## Getting Started

To get started with development, you'll need to set up a virtual environment and install the project in editable mode.

1.  **Create and activate a virtual environment:**
    ```bash
    python3 -m venv venv
    source venv/bin/activate
    ```

2.  **Install the project in editable mode:**
    ```bash
    pip install -e .
    ```

This will install all the necessary dependencies and make the `npm-scan` command available in your shell.

## Running Tests

Currently, there are no automated tests for this project. If you add a new feature, please also add tests for it.

To run the tests (when they are added), you can use the following command:

```bash
# Placeholder for test command
pytest
```

## Submitting Changes

When you're ready to submit your changes, please make sure to:

-   **Provide a clear and descriptive pull request title and description.**
-   **Link to any relevant issues.**
-   **Ensure your code follows the existing code style.**
-   **Update the documentation** if you've added or changed any features.

## Code of Conduct

This project and everyone participating in it is governed by the [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior to the project maintainers.
